ChimeraHydra

两个独立的 HydraLoRA（Tian et al., NeurIPS 2024, arXiv:2404.19245）在残差处拼合 — 这就是"奇美拉"。由 HydraLoRA 的逐层秩-R 路由器基于池化文本特征路由的内容池（K_c=3 个 B-头），加上由网络级 FreqRouter 基于频率能量指数 FEI(z_t) + 正弦-σ 特征路由的频率池（K_f=3 个 B-头，FeRA, arXiv:2511.17979）。两个池的输出直接相加 — 没有乘法门控，没有 σ-频带重叠掩码。专门化通过路由器输入分离来强制实现：内容路由器从不接收 σ，频率路由器从不接收文本。

每个 Linear 在基础权重的不相交 SVD 子空间上（将内容/频率拆分的前-2r 右奇异向量）携带两个 Cayley 参数化的 A 因子，在下行侧和上行侧均提供自由正交性。T-LoRA 的秩掩码（TimeStep Master, arXiv:2503.07416）仅调制内容分支 — 频率分支在所有 t 下保持全秩，形成不对称的"核心专家始终激活 / 上下文专家秩调制"分割。

属于 LoRA 系列网络模块；设置 use_chimera_hydra=true 会自动将三轴路由面锁定为 shared_A / route_per_layer=true / router_source="input"，并将其戳印到保存元数据中，使加载器无需并行判别路径即可重新实例化奇美拉。两个池、两个路由器和两个共享 A 在去噪损失 + 每池均衡损失下单阶段联合训练。

训练生成 anima_chimera*.safetensors + 包含路由器的 *_moe.safetensors 附属文件；路由器实时推理需要两个文件（make test-hydra / ComfyUI Anima Adapter Loader）。需要 cache_llm_adapter_outputs=true。参见 docs/proposal/chimera_hydra.md 和 networks/lora_modules/chimera.py。
