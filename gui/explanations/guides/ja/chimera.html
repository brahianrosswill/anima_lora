ChimeraHydra

残差で結合された 2 つの独立した HydraLoRA (Tian et al., NeurIPS 2024, arXiv:2404.19245) — これがキメラです。プールされたテキスト特徴を入力とする HydraLoRA の per-layer ランク-R ルーターでルーティングされる コンテンツプール (K_c=3 B-ヘッド)と、周波数エネルギーインデックス FEI(z_t) + 正弦波-σ 特徴 (FeRA, arXiv:2511.17979) を入力とするネットワークレベルの FreqRouter でルーティングされる 周波数プール (K_f=3 B-ヘッド)で構成されます。2 つのプール出力は加算されます — 乗算ゲートも σ-バンドオーバーラップマスクもありません。専門化はルーター入力の分離によって強制されます: コンテンツルーターは σ を参照せず、周波数ルーターはテキストを参照しません。

各 Linear はベース重みの分離した SVD 部分空間 (コンテンツ/周波数を分割する上位-2r 右特異ベクトル) 上に 2 つの Cayley パラメタライズされた A ファクターを持ち、ダウン側とアップ側の両方で自由な直交性を提供します。T-LoRA のランクマスク (TimeStep Master, arXiv:2503.07416) はコンテンツブランチのみを変調します — 周波数ブランチはすべての t でフルランクを維持する非対称な「コアエキスパート常時オン / コンテキストエキスパートランク変調」の分割です。

LoRA ファミリーネットワークモジュールに属します。use_chimera_hydra=true を設定すると、三軸ルーティングサーフェスが自動的に shared_A / route_per_layer=true / router_source="input" に固定され、保存メタデータにスタンプされることでローダーが並列識別パスなしにキメラを再インスタンス化します。両プール、両ルーター、両共有 A はすべてデノイズ損失 + プールごとのバランス損失の下で単一フェーズとして同時に学習されます。

学習すると anima_chimera*.safetensors + ルーターを保持する *_moe.safetensors サイドカーが生成されます。ルーターライブ推論には両ファイルが必要です (make test-hydra / ComfyUI Anima Adapter Loader)。cache_llm_adapter_outputs=true が必要です。docs/proposal/chimera_hydra.md および networks/lora_modules/chimera.py を参照してください。
