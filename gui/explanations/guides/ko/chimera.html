ChimeraHydra

잔차에서 합쳐진 두 개의 독립적인 HydraLoRA (Tian et al., NeurIPS 2024, arXiv:2404.19245) — 이것이 키메라입니다. 풀링된 텍스트 피처를 입력으로 받는 HydraLoRA의 레이어별 랭크-R 라우터로 라우팅되는 콘텐츠 풀 (K_c=3개 B-헤드)과, 주파수-에너지 지수 FEI(z_t) + 정현파-σ 피처 (FeRA, arXiv:2511.17979)를 입력으로 받는 네트워크 수준의 FreqRouter로 라우팅되는 주파수 풀 (K_f=3개 B-헤드)로 구성됩니다. 두 풀의 출력은 더해집니다 — 곱셈 게이트도, σ-밴드 오버랩 마스크도 없습니다. 전문화는 라우터 입력 분리를 통해 강제됩니다: 콘텐츠 라우터는 σ를 절대 보지 않으며, 주파수 라우터는 텍스트를 절대 보지 않습니다.

각 Linear는 베이스 가중치의 분리된 SVD 부분공간 (콘텐츠/주파수를 분리하는 상위-2r 오른쪽 특이벡터) 위에 두 개의 Cayley-파라미터화된 A 팩터를 가지며, 다운 측과 업 측 모두에서 자유로운 직교성을 제공합니다. T-LoRA의 랭크 마스크 (TimeStep Master, arXiv:2503.07416)는 콘텐츠 브랜치에만 적용됩니다 — 주파수 브랜치는 모든 t에서 풀랭크를 유지하는 비대칭적인 "핵심 전문가는 항상 활성 / 컨텍스트 전문가는 랭크 변조" 분할입니다.

LoRA 계열 네트워크 모듈에 속합니다. use_chimera_hydra=true를 설정하면 3축 라우팅 서페이스가 자동으로 shared_A / route_per_layer=true / router_source="input"으로 고정되고, 이 정보가 저장 메타데이터에 스탬프되어 로더가 별도의 판별 경로 없이 키메라를 재인스턴스화합니다. 두 풀, 두 라우터, 두 공유 A 모두 단일 단계에서 디노이즈 손실 + 풀별 균형 손실로 함께 학습됩니다.

학습 시 anima_chimera*.safetensors + 라우터를 담은 *_moe.safetensors 동반 파일이 생성됩니다. 라우터-라이브 추론에는 두 파일 모두 필요합니다 (make test-hydra / ComfyUI Anima Adapter Loader). cache_llm_adapter_outputs=true 필요. docs/proposal/chimera_hydra.md 및 networks/lora_modules/chimera.py를 참조하세요.
