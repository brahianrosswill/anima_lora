Colorize

An EasyControl control task: colorize a black & white screentoned manga page. Same frozen-DiT EasyControl network as the base variant, but the condition is a synthetic mangafied B&W render (XDoG edges + screentone) and the captions are color-only (hue / saturation words).

• Empty prompt → auto-colorize (the model picks plausible colors).
• Color prompt → steers the palette (e.g. "blue hair, red dress").

The color-only text cache is multi-variant (full set + shuffled with ~30% of color tags dropped) so partial color prompts still work; the manga cond is a lossy hint, so training-time cond perturbation is kept small to preserve line structure. Cond latents cache to post_image_dataset/colorize_cond/.

Train from the EasyControl tab — it sets EASYADAPTER=colorize — and preprocess runs the mangafy filter before VAE-encoding the condition. See easycontrol_adapters/colorization/ and docs/experimental/easycontrol.md.
