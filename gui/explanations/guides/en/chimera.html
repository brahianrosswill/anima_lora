ChimeraHydra

Two independent HydraLoRAs (Tian et al., NeurIPS 2024, arXiv:2404.19245) glued at the residual — that's the chimera. A content pool (K_c=3 B-heads) routed by HydraLoRA's per-layer rank-R router on pooled text features, plus a freq pool (K_f=3 B-heads) routed by a network-level FreqRouter fed the frequency-energy index FEI(z_t) + sinusoidal-σ features (FeRA, arXiv:2511.17979). The two pool outputs are added — no multiplicative gate, no σ-band overlap mask. Specialization is enforced by router-input separation: the content router never sees σ, the freq router never sees text.

Each Linear carries two Cayley-parameterized A factors off disjoint SVD subspaces of the base weight (top-2r right-singular vectors split content/freq), giving free orthogonality on both the down and up sides. T-LoRA's rank mask (TimeStep Master, arXiv:2503.07416) modulates the content branch only — the freq branch stays full-rank at every t, an asymmetric "core expert always on / context expert rank-modulated" split.

Lives on the LoRA-family network module; setting use_chimera_hydra=true auto-pins the three-axis routing surface to shared_A / route_per_layer=true / router_source="input" and stamps it into the save metadata so the loader re-instantiates the chimera without a parallel discrimination path. Both pools, both routers, and both shared A's train jointly under the denoise loss + per-pool balance loss (single phase).

Training produces anima_chimera*.safetensors + a *_moe.safetensors sibling holding the routers; both files are needed for router-live inference (make test-hydra / ComfyUI Anima Adapter Loader). Requires cache_llm_adapter_outputs=true. See docs/proposal/chimera_hydra.md and networks/lora_modules/chimera.py.
