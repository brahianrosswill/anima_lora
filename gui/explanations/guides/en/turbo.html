Turbo (Decoupled DMD2)

Few-step distillation via Decoupled-Hybrid DMD2 (Liu et al., arXiv:2511.22677). Distills the 28-step CFG=4 Anima teacher into a 4-step student, with three roles — student, fake-score, and teacher — sharing one frozen DiT. CFG is baked into the student during distillation.

The output is a plain standard LoRA. There is no inference-side turbo code: you load it through the normal LoRA adapter path and run --infer_steps 4 --cfg 1.0 (make exp-test-turbo). It composes with concept LoRAs the way LCM-LoRA composes with style LoRAs (linear composition, ranks add), and is fully bakeable into the DiT.

Training runs through scripts/distill_turbo.py — a bespoke single-GPU loop (like distill-mod / distill-spd), not train.py. Launch with make exp-turbo (honors PRESET; append overrides via ARGS=…).

Config is the odd one out. configs/methods/turbo.toml is a bespoke sectioned schema read only by the script — the flat method+preset merge does not apply, so don't print-config METHOD=turbo. CLI flags override TOML values. See docs/experimental/dmd2-decoupled.md (ops) and docs/structure/dmd2-decoupled.md (math).
